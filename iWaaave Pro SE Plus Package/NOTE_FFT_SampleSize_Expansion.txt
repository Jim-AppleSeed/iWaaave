FFT Expansion: 16 samples -> 256 samples

Conceptual Explanation
Why 256 Points?
A 16-point FFT at 48kHz sampling rate gives you only 8 unique frequency bins (N/2), each spanning 3kHz (48000/16). That's extremely coarse — you can barely distinguish bass from treble. A 256-point FFT gives you 128 unique frequency bins, each spanning 187.5 Hz (48000/256). This lets you meaningfully distinguish between bass, midrange, and treble frequencies.

What Changes Architecturally
1. Sample Buffer: Must collect 256 samples instead of 16. At 48kHz, this takes ~5.3ms per frame (~188 FPS worth of data), still plenty fast for display.

2. FFT Engine: Goes from 4 stages (log2(16)=4) to 8 stages (log2(256)=8). Each stage has 128 butterflies (N/2=128) instead of 8. The old approach of doing one butterfly per clock cycle would take 8×128=1024 cycles — still well under 1ms at 50MHz, so the sequential approach remains viable.

3. Twiddle Factors: We need W_256^k for k=0..127 instead of W_16^k for k=0..7. That's 128 complex constants instead of 8. These are stored in ROM.

4. Bit-Reversal: Changes from 4-bit reversal to 8-bit reversal.

5. Butterfly Indexing: The same formula applies but with 8-bit indices. For stage s (1-8): block_size=2^s, half=2^(s-1), and we iterate through 128 butterflies per stage.

6. Magnitude-to-Display Mapping: With 128 meaningful frequency bins (bins 1-127) but only 16 LED columns, we must bin the frequencies. I use a logarithmic-like grouping so that lower frequencies (which humans are more sensitive to distinguishing) get individual columns while higher frequencies are grouped together. This mirrors how audio spectrum analyzers work in practice.

Twiddle Factor Generation
For a 256-point FFT: W_256^k = cos(2πk/256) - j·sin(2πk/256), for k = 0 to 127. These are pre-computed in Q1.14 fixed-point format and stored in a ROM that is initialized via a .hex file generated by a Python script.

Frequency Bin Grouping for Display
The 128 bins are mapped to 16 columns using these groups:

Columns 0-7: Individual bins (fine resolution for low frequencies, ~187.5 Hz each)
Columns 8-11: Groups of 4 bins each
Columns 12-13: Groups of 8 bins each
Columns 14-15: Groups of 16 bins each
This gives perceptually meaningful frequency separation on a 16-column display.
